src/ingestion/parse.py
─────────────────────────────────────────────────────────────────────────────────

Parses PubMed Central JATS XML into structured Python objects.

Key data classes:
  Section
    name          — canonical label (abstract | introduction | methods |
                    results | discussion | conclusion | other)
    raw_name      — original heading text from XML
    content       — cleaned plain text
    section_type  — "text" | "table" | "figure_caption"

  ParsedPaper
    pmc_id, pmid, doi, title, abstract, journal, year, authors, sections

Public API:
  parse_paper(xml_str) -> ParsedPaper | None
    Parses a JATS XML string. Returns None if malformed, missing <front>,
    or missing title. Abstract is always prepended as the first section.
    References, acknowledgements, funding, and similar boilerplate sections
    are hard-skipped during the recursive walk.

  normalize_section(heading) -> str
    Maps any raw section heading to a canonical label or "_skip".
    Driven by _SECTION_MAP — a 50+ entry list of (keyword, canonical) tuples
    checked longest-first so multi-word phrases ("materials and methods")
    match before single keywords ("methods").

Implementation details:
  _walk()          — recursively processes <sec> elements; extracts <p> text,
                     <table-wrap>, and <fig> captions as separate Section objects;
                     subsections inherit the parent canonical label if their own
                     heading is unrecognised
  _table_text()    — renders caption + thead rows + tbody rows as pipe-delimited text
  _figure_caption() — label + caption text joined with ": "
  _get_text()      — recursive XML text extraction (handles mixed content)
  _clean()         — strips inline citation markers [1], [1,2], [1-3], [1–3];
                     collapses all whitespace to a single space
  _local()         — strips XML namespace prefix so tags work with or without
                     namespace declarations
  _extract_year()  — parses <year> element to Optional[int]

Bug fixed during testing:
  "Supplemental Data" was returning "other" instead of "_skip" because only
  "supplementary" was in _SECTION_MAP. Added ("supplemental", "_skip") immediately
  after the "supplementary" entry to cover both spellings.

src/ingestion/chunk.py
─────────────────────────────────────────────────────────────────────────────────

Converts a ParsedPaper into a flat list of Chunk objects with full metadata.

Constants:
  SHORT_SECTION_THRESHOLD = 80    words; sections at or below → single chunk
  TABLE_WORD_CAP          = 500   words; tables longer than this are truncated
  TABLE_TRUNCATION_NOTE   = "[Table truncated for length]"
  EVIDENCE_LEVELS = {meta_analysis:1, rct:2, cohort:3, case_report:4,
                     review:5, unknown:6}

Key data classes:
  ChunkMetadata — 14 fields: pmid, pmcid, title, authors, journal, year,
                  cancer_type, section, chunk_type, chunk_index,
                  study_design, sample_size, primary_outcome, evidence_level
  Chunk         — id (stable string), text, metadata (ChunkMetadata)

Stable chunk ID format: "{pmcid}_{section}_{chunk_index}"
  chunk_index is a global counter across all sections of a paper, so IDs are
  unique within a paper and deterministic across runs.

Public API:
  chunk_paper(paper, cancer_type, study_design="unknown", sample_size=None,
              primary_outcome=None, chunk_size=200, overlap=30) -> list[Chunk]
    Main entry point. Builds a base metadata dict, iterates paper.sections,
    skips empty/whitespace-only sections, accumulates a global chunk index.

  chunk_section(section, base_metadata, start_index, chunk_size=200,
                overlap=30) -> list[Chunk]
    Dispatches to one of three strategies:
      • Tables        → _table_chunk()  (always 1 chunk, truncated if over cap)
      • Figure captions → _single()    (always 1 chunk)
      • Short text    → _single()      (≤ SHORT_SECTION_THRESHOLD words)
      • Long text     → _sliding()     (overlapping sliding window)

  _sliding() — step = chunk_size - overlap; while loop advances by step;
               terminates when pos + chunk_size >= len(words) to avoid a
               trailing empty/duplicate chunk.

src/ingestion/extract_metadata.py
─────────────────────────────────────────────────────────────────────────────────

Extracts three fields that cannot be reliably parsed from XML alone:
  study_design   — one of: rct | meta_analysis | cohort | case_report |
                   review | unknown
  sample_size    — integer or null
  primary_outcome — one sentence or null

Uses OpenAI structured outputs (response_format with json_schema type).
Requires openai >= 1.40.0.

Key data class:
  ExtractionResult — pmid, study_design, sample_size, primary_outcome,
                     extraction_failed (bool), extraction_model (str)

Public API:
  extract_metadata(pmid, abstract, openai_client, cache_path, model) -> ExtractionResult
    Checks data/metadata_cache.json first (keyed by pmid).
    On cache miss: calls LLM, writes result back to cache.
    Empty abstract → returns extraction_failed=True without calling LLM.
    All LLM/network exceptions are caught → returns extraction_failed=True.
    Cache write failures are non-fatal (silent pass).

OpenAI call details:
  model:            gpt-4o-mini (default)
  temperature:      0.0
  max_tokens:       150
  response_format:  {"type": "json_schema", "json_schema": {"name":
                    "medical_study_metadata", "strict": True, "schema": ...}}
  System prompt instructs: "Extract ONLY information explicitly stated.
                            Do NOT infer or assume."

tests/unit/test_chunking.py  (46 tests, all passing)
─────────────────────────────────────────────────────────────────────────────────

TestNormalizeSection   (11 tests)
  Covers all 6 canonical section labels, _skip for references/acknowledgements/
  funding/supplemental, "other" for unknown headings and empty string,
  and case-insensitivity.

TestShortSection       (6 tests)
  Single chunk for 1 word, below threshold, and at threshold.
  Full text preserved. Empty and whitespace-only content → [].

TestTableChunks        (7 tests)
  Tables are always 1 chunk regardless of size. chunk_type == "table".
  Over-cap tables are truncated with TABLE_TRUNCATION_NOTE appended.
  Truncated word count bounded at TABLE_WORD_CAP + 10.
  Figure captions are 1 chunk with chunk_type == "figure_caption".

TestChunkIds           (5 tests)
  ID format: "PMC000001_methods_7". Stable across repeated calls.
  Unique within a multi-section paper. start_index offset is reflected.
  Global chunk_index is monotonically increasing and zero-based.

TestOverlap            (5 tests)
  Uses chunk_size=40, overlap=8, n_words=100 (above 80-word threshold).
  Multiple chunks produced. Last 8 words of chunk N == first 8 words of
  chunk N+1 at every boundary. No words lost. No chunk exceeds chunk_size.
  Higher overlap → more chunks than lower overlap.

TestMetadataFields     (5 tests)
  All 14 required ChunkMetadata fields present. Evidence level derived
  correctly for every study_design value. Section label, cancer_type,
  and chunk_type all propagated correctly.

TestChunkPaper         (7 tests)
  Empty paper → []. Single short section → 1 chunk. cancer_type,
  study_design, and evidence_level passed through from caller.
  Unknown study_design defaults to evidence_level=6. Multi-section chunk
  count correct. Whitespace-only sections are skipped.
