src/db/vector_store.py
─────────────────────────────────────────────────────────────────────────────────

Qdrant wrapper. All other modules import from here; nothing imports qdrant-client
directly.

Key data classes:
  ChunkDocument — input to upsert; contains dense_vector, sparse_indices,
                  sparse_values, and all 14 metadata fields from ChunkMetadata.
  ScoredChunk   — output of all search methods; chunk_id, text, score, metadata.

Collection: urological_oncology_v2
  Dense vector:  "dense", 1536-dim cosine (text-embedding-3-small)
  Sparse vector: "sparse", hash-based TF weights (SparseVectorParams)
  Payload indexes on: cancer_type, section, study_design, chunk_type (KEYWORD),
                      year, evidence_level (INTEGER)

Public methods:
  upsert(chunks, batch_size=100)       — batched PointStruct upsert
  search_dense(embedding, top_k, filters) — Qdrant HNSW ANN search
  search_sparse(tokens, top_k, filters)   — Qdrant sparse dot-product search
  scroll_all(batch_size=500)           — full collection scan for BM25 indexing
  delete_by_pmid(pmid)                 — FilterSelector delete
  collection_stats()                   — point count + collection info dict
  ensure_collection()                  — create collection + indexes if absent

Sparse vector helpers (importable by ingestion pipeline):
  text_to_sparse_vector(text) -> (indices, values)
    Hash vocabulary: token → abs(hash(token)) % 2^20.  TF weights.  Indices
    sorted ascending (Qdrant requirement).  Collisions summed.
  tokens_to_sparse_query(tokens) -> (indices, values)
    Binary weights (1.0 per unique token); used in search_sparse.

Filter dict keys (same across all retrieval modules):
  cancer_type, section, study_design, chunk_type — list[str], match any
  year_min, year_max                              — int, range
  evidence_level_max                             — int, upper bound

ID strategy: Qdrant point ID = uuid5(NAMESPACE_DNS, chunk_id) — deterministic
UUID derived from the stable string chunk_id.

src/retrieval/bm25_search.py
─────────────────────────────────────────────────────────────────────────────────

In-memory BM25 index using rank_bm25 (BM25Okapi).  Complements dense search by
excelling at exact clinical term matching: drug names, gene symbols, numeric
thresholds.

Construction:
  BM25Search(chunks: list[ScoredChunk])    — direct from a chunk list
  BM25Search.from_qdrant(store)            — loads all chunks via scroll_all()

Public methods:
  build(chunks)    — rebuild the index (called after large ingestion runs)
  search(query, top_k, filters=None) -> list[ScoredChunk]
  size             — property: number of indexed chunks

Tokeniser (_tokenize):
  Lowercases, strips all punctuation via regex, splits on whitespace.

Bug fixed during development (discovered by test_build_replaces_index):
  In small corpora all BM25 IDF values are negative (a term appearing in every
  document gets IDF < 0 by the Okapi formula). The original code broke the
  search loop at score <= 0.0, which incorrectly excluded relevant documents
  whose score happened to be negative.
  Fix: filter using np.nonzero(scores) — excludes only documents where score
  is exactly 0.0 (no query terms present), then sorts the non-zero subset
  descending.  Documents with negative BM25 scores (matching terms, negative
  IDF) are correctly included and ranked.

Filter application: post-scoring on stored ScoredChunk.metadata dict.
  Supports same filter keys as QdrantStore.

src/retrieval/hybrid.py
─────────────────────────────────────────────────────────────────────────────────

Pure function: rrf_fusion(dense_results, bm25_results, k=60, top_k=None)

RRF formula:
  score(chunk) = 1 / (k + rank_dense) + 1 / (k + rank_bm25)

Rules:
  Ranks are 1-based.
  Chunk absent from one list → fallback rank = max(len_dense, len_bm25) + 1.
  All chunk IDs are deduplicated (union of both lists).
  Output sorted descending by RRF score.
  top_k=None returns all merged chunks.

Metadata and text are preserved from whichever list first supplied the chunk.

src/retrieval/reranker.py
─────────────────────────────────────────────────────────────────────────────────

Key data class:
  RankedChunk — chunk_id, text, score (final), relevance_score (raw Cohere),
                metadata.

CohereReranker:
  Uses cohere.ClientV2 (cohere >= 5.0 API).
  Model: rerank-english-v3.0.
  Constructed with api_key=""; is_available() returns False → passthrough mode.

Final score formula (study-design weighted):
  final_score = relevance_score^0.8 * study_design_weight^0.2
  Gives mild preference to higher-evidence sources at equal relevance.
  study_design_weight looked up from config.constants.STUDY_DESIGN_WEIGHTS.

Graceful degradation:
  api_key empty OR any API exception → returns first top_n input chunks as
  RankedChunk with relevance_score = original RRF score.
  Reranker latency is logged at INFO level.

src/retrieval/retriever.py
─────────────────────────────────────────────────────────────────────────────────

Key data class:
  RetrievalResult — query, chunks (list[RankedChunk]), retrieval_confidence,
                    num_candidates, latency_ms (dict with per-step timings).

RAGRetriever (constructed once at startup, injected as FastAPI dependency):
  __init__ takes: QdrantStore, BM25Search, CohereReranker, OpenAI client,
                  embedding_model, top_k_retrieval=20, top_k_rerank=5.

Pipeline in retrieve(query, filters, top_k_retrieval, top_k_rerank):
  1. Embed query via OpenAI (in-process dict-based LRU cache by (query, model))
  2. Dense search — filters applied natively in Qdrant (most efficient)
  3. BM25 search — filters applied on in-memory corpus
  4. RRF fusion → top_k_retrieval merged candidates
  5. Cohere rerank → top_k_rerank final chunks
  6. retrieval_confidence = mean(relevance_score) across returned chunks

Metadata filtering always before reranking as specified.

tests/integration/conftest.py
─────────────────────────────────────────────────────────────────────────────────

Session-scoped fixtures (no live APIs, no external services):

  qdrant_store   — QdrantClient(":memory:") wrapped in QdrantStore
  indexed_chunks — Parses all 10 XML fixtures, chunks each paper via
                   chunk_paper(), assigns deterministic embeddings, upserts
                   to in-memory Qdrant, returns list[ScoredChunk] for BM25.
  bm25_index     — BM25Search built from indexed_chunks.

Deterministic embeddings (make_embedding):
  90% weight on a cancer-type basis vector (fixed numpy unit vector seeded by
  cancer type name) + 10% random noise seeded by chunk_id.
  Same-cancer-type documents cluster together; queries using the pure basis
  vector (query_embedding_for) reliably retrieve same-cancer chunks over
  cross-cancer chunks.
