src/observability/logging.py
─────────────────────────────────────────────────────────────────────────────

JSON-formatted structured logging for log aggregation platforms.

Context vars (module-level, used across the codebase):
  request_id_var: ContextVar[str]   — UUID set by request-ID middleware
  query_id_var:   ContextVar[str]   — UUID set per RAG query in the route handler

_JsonFormatter
  Formats every record as a JSON object with: timestamp (ISO-8601 UTC),
  level, logger, message, request_id, query_id.  Includes "exception" key
  when exc_info is present.

RequestContextFilter
  logging.Filter subclass — injects request_id and query_id from context
  vars into each LogRecord so they are available for non-JSON formatters too.

setup_logging(level="INFO")
  Clears existing root handlers and replaces with a single StreamHandler
  using _JsonFormatter + RequestContextFilter.

get_logger(name) → logging.Logger
  Thin wrapper around logging.getLogger — use everywhere instead of direct
  calls to keep the import path consistent.

Bug fixed: removing duplicate index on Chunk.pmc_id.
  mapped_column(ForeignKey(...), index=True) auto-names the index

src/observability/audit.py
─────────────────────────────────────────────────────────────────────────────

INSERT-only audit log backed by synchronous SQLAlchemy.

AuditLogger(db_url: str)
  Creates a sync Engine from db_url (SQLite or PostgreSQL).
  sqlite:// URLs get connect_args={"check_same_thread": False}.
  Calls Base.metadata.create_all() at construction — safe for SQLite test DBs.

async log(query_id, question, result, retrieval_result, confidence, gate,
          user_id=None, session_id=None) → None
  Builds sources JSON from retrieval_result.chunks.
  Computes total latency = sum(retrieval latency) + generation latency.
  Delegates to _insert() via asyncio.to_thread() (non-blocking).

_insert(**kwargs) → None  (synchronous, called via to_thread)
  Opens a Session, adds AuditLog row, commits.  Never updates or deletes.

src/observability/tracing.py
─────────────────────────────────────────────────────────────────────────────

Langfuse distributed tracing with complete graceful degradation.

If the langfuse package is not installed OR LANGFUSE_PUBLIC_KEY is empty:
  - setup_tracing() is a no-op.
  - trace_query() yields a QueryTrace backed by None.
  - All span/score/set_metadata calls are no-ops.
  Application code never needs try/except around tracer calls.

setup_tracing(settings) → None
  Initialises the module-level _client = langfuse.Langfuse(...).

QueryTrace
  span(name, **kwargs) — @contextmanager; creates a real span if _trace is
    set, yields _NullSpan otherwise.  Calls span.end() in finally.
  score(name, value, comment) — no-op if no trace.
  set_metadata(**kwargs) — calls trace.update(metadata=...) if trace is set.

@contextmanager trace_query(query_id, question) → Iterator[QueryTrace]
  Opens Langfuse trace (or yields null trace), flushes client on exit.

src/api/middleware/auth.py
─────────────────────────────────────────────────────────────────────────────

FastAPI Depends function (not Starlette middleware) so Swagger UI stays
accessible in development.

require_api_key(request, settings) → str
  Exempt paths: /health/live, /health/ready (always return "").
  Dev bypass: returns "dev" when app_env == "development" AND api_keys == [].
  Otherwise: reads the header named by settings.api_key_header, raises
    HTTP 401 "Missing API key" if absent, HTTP 401 "Invalid API key" if
    the value doesn't match any key in settings.api_keys.
  Comparison uses hmac.compare_digest to prevent timing attacks.

src/api/middleware/rate_limit.py
─────────────────────────────────────────────────────────────────────────────

Sliding-window in-process rate limiter (BaseHTTPMiddleware).
TODO: Replace _counters dict with Redis for multi-instance deployments.

Exempt paths: any path starting with /health.

Per request:
  _resolve_key(request) → (counter_key, limit)
    Authenticated (X-API-Key header present): key = "key:{api_key}", limit = settings.rate_limit_per_minute
    Anonymous (no header): key = "ip:{client_host}", limit = 10 (_ANON_LIMIT)

  Window: 60 seconds (_WINDOW_SECONDS). If now - window_start >= 60s,
    counter resets.

  Under limit: increment counter, add X-RateLimit-{Limit,Remaining,Reset}
    headers to response.
  At/over limit: return 429 JSON {"detail": "Rate limit exceeded"} with
    Retry-After, X-RateLimit-Limit=limit, Remaining=0, Reset headers.

src/api/routes/query.py
─────────────────────────────────────────────────────────────────────────────

POST /query — main RAG endpoint.

Pydantic models:
  QueryRequest   — query (1–500 chars), cancer_types, year_from, year_to,
                   study_designs, conversation_id, top_k (1–10, default 5),
                   stream (bool)
  SourceCard     — chunk_id, title, authors, journal, year, study_design,
                   sample_size, section, key_finding (first 150 chars), pmid
  LatencyBreakdown — retrieval, rerank, generation, total (all int ms)
  QueryResponse  — answer, evidence_quality, confidence_score, sources,
                   conversation_id, request_id, latency_ms

Dependency accessors (overridable in tests):
  get_retriever(request)   → app.state.retriever
  get_generator(request)   → app.state.generator
  get_audit_logger(request) → app.state.audit_logger

Pipeline in query_endpoint():
  1. Build filters dict from cancer_types / study_designs / year_from / year_to.
  2. retriever.retrieve(query, filters, top_k_rerank=top_k)
  3. generator.generate(query, ranked_chunks)
  4. Audit log (fire-and-forget — exception is logged as WARNING, not raised).
  5. Build SourceCard list from retrieval_result.chunks.
  6. If stream=True → return StreamingResponse with single SSE event.
  7. Otherwise → return QueryResponse.

Error codes: 503 if retriever/generator is None (not initialised).

src/api/main.py
─────────────────────────────────────────────────────────────────────────────

create_app() → FastAPI  (module-level `app = create_app()` for uvicorn)

Middleware stack (outermost first):
  1. CORSMiddleware — allow_origins=["*"] in development, [] in production.
  2. RateLimitMiddleware — using settings.rate_limit_per_minute.
  3. request_id_middleware (@app.middleware("http")) — generates UUID per
     request, sets request_id_var context var, adds X-Request-ID header.
  4. Global exception handler — catches all unhandled exceptions, logs them,
     returns {"detail": "An internal error occurred"} (no tracebacks exposed).

Lifespan (asynccontextmanager):
  Startup:
    setup_logging, setup_tracing
    Try Qdrant → BM25 → CohereReranker → OpenAI → RAGRetriever (→ state.retriever)
    Try LLMClient → ClinicalGenerator (→ state.generator)
    Try AuditLogger (→ state.audit_logger)
    Each component logs WARNING on failure but does NOT abort startup.
  Shutdown: logs "Application shutting down".

Routers: query.router, health.router, eval.router.

BUGS FIXED DURING IMPLEMENTATION
─────────────────────────────────────────────────────────────────────────────

1. Duplicate index ix_chunks_pmc_id in models.py
   Cause: mapped_column(ForeignKey(...), index=True) auto-creates
   ix_chunks_pmc_id; an explicit Index("ix_chunks_pmc_id") in __table_args__
   then tried to create it again.
   Fix: removed the explicit Index() from __table_args__.

2. Hallucinated-citation test assertion
   Cause: the WARNING header itself says "([Doc 9]) that were removed",
   so `"[Doc 9]" not in result.answer` always failed.
   Fix: split on "\n\n" and check only the answer body portion:
   `assert "[Doc 9]" not in result.answer.split("\n\n", 1)[-1]`
