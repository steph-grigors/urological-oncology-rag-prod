─────────────────────────────────────────────────────────────
KEY DESIGN DECISIONS
─────────────────────────────────────────────────────────────

ADMIN_API_KEY fallback:
  When ADMIN_API_KEY is empty the admin routes fall back to the regular
  API_KEYS check. This means development stacks with only API_KEYS set
  can still trigger eval runs without a separate admin key.

conversation_id as UUID string:
  Server-side conversation tracking — the UI only stores the UUID;
  conversation history lives in the backend. Enables stateless UI
  containers that can restart without losing conversation context.

Evidence quality badge:
  Computed by the generation layer based on reranker confidence and
  source agreement. Displayed in the results header so clinicians
  immediately see reliability signal before reading the answer.

Background eval with run_in_executor:
  run_evaluation() is a synchronous function (file I/O, blocking LLM
  calls). Wrapping it in run_in_executor keeps the FastAPI event loop
  free while the eval run is in progress.
