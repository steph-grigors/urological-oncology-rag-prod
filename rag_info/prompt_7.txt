src/evaluation/golden_set.py
─────────────────────────────────────────────────────────────────────────────

GoldenQuery (dataclass)
  id, query, cancer_type, difficulty, question_type,
  expected_study_designs, must_contain_terms, must_not_hallucinate,
  ground_truth, expected_sources, notes

GoldenSet (dataclass)
  version: str; queries: list[GoldenQuery]

load_golden_set(path) → GoldenSet
  Reads JSON and maps both v1 (question/topic) and v2 (query/cancer_type)
  schemas into GoldenQuery objects.

add_query(set_path, query) → None
  Appends a query to the JSON file, auto-assigning an incremental G-ID
  if query.id is empty. Creates the file if it doesn't exist.

src/evaluation/judges.py
─────────────────────────────────────────────────────────────────────────────

JudgeScores (dataclass)
  faithfulness, answer_relevance, context_precision, context_recall,
  clinical_safety, citation_accuracy, evidence_appropriate
  overall: float — recomputed in __post_init__ as mean of first 5
  rationales: dict[str, str] — explanation per metric

JudgeSet(openai_client=None, model="gpt-4o-mini")
  score_all(question, answer, chunks, ground_truth=None) → JudgeScores
    Routes to _heuristic_score_all (no API) or _llm_score_all (one
    JSON-object LLM call covering all metrics, heuristic fallback).

Heuristic judges (all return (float, rationale_str)):
  _heuristic_faithfulness(answer, chunks)
    No citations → 0.85. All [Doc N] in range → 1.0. Penalty per invalid.
  _heuristic_answer_relevance(question, answer)
    Key-term overlap: matched / total_key_terms (stopwords + len≤3 filtered).
  _heuristic_context_precision(question, chunks)
    relevant_chunks / total_chunks (chunk text contains any query key term).
  _heuristic_context_recall(answer, ground_truth)
    1.0 if ground_truth is None/"". Otherwise GT key-term overlap in answer.
  _heuristic_clinical_safety(answer)
    1.0 if no directive patterns. -0.5 per pattern found.
  _heuristic_citation_accuracy(answer, chunks)
    Valid [Doc N] references (N in range and chunk text non-empty).
  _heuristic_evidence_appropriate(answer, chunks)
    1.0 if all sources high-evidence (level<4) or answer contains hedging.
    0.7 if low-evidence sources used without hedging.

src/evaluation/runner.py
─────────────────────────────────────────────────────────────────────────────

QueryResult (dataclass)
  query_id, query, cancer_type, difficulty, question_type, answer,
  scores: JudgeScores, latency_ms, contains_required_terms, is_refused

EvaluationReport (dataclass)
  run_id, timestamp, golden_set_version, mode, total_queries,
  aggregate: JudgeScores, per_topic, per_difficulty, per_question_type,
  latency_stats (p50/p95/p99/mean), regression_delta, passed_regression,
  per_query_results: list[QueryResult]

run_evaluation(retriever, golden_set_path, output_dir, mode,
               baseline_path, max_workers, generator, judge_set)
  → EvaluationReport
  ThreadPoolExecutor over all golden queries. output_dir=None skips file
  writes (used in tests). Saves {ts}_detailed.json, {ts}_metrics.json,
  latest_metrics.json if output_dir is set.

Modes: full (all 30 queries), quick (random 10), regression (baseline diff).
Regression tolerance: -0.02 pp on any of the 5 primary metrics.
Exit-code contract for CLI use: faithfulness < 0.80 or relevance < 0.75
triggers sys.exit(1).
